# پرامپت ساخت CRM آرکان (فاز ۴) — برای Claude Code

> این پرامپت را در ریشه‌ی پروژه‌ی «وب‌سایت + چت‌بات آرکان» (فاز ۱ و ۲) به Claude Code بدهید.
> نتیجه: ماژول CRM کاملی که در همین ریپو می‌بینید — با همان معماری، همان قراردادهای کد و همان برند.
>
> 🎓 نکته برای دانشجو: به ساختار پرامپت دقت کنید — «زمینه‌ی موجود»، «تصمیم‌های قطعی»،
> «الزامات هر ماژول» و «معیار پذیرش». پرامپت خوب یعنی تصمیم‌های محصولی را شما بگیرید
> و پیاده‌سازی را به مدل بسپارید؛ نه برعکس.

---

برای شرکت مشاوره‌ی کسب‌وکار «آرکان» می‌خواهم روی همین پروژه‌ی موجود (Next.js 14 App Router + Supabase + Tailwind، فارسی و RTL) یک **CRM کامل مبتنی بر best practiceهای صنعت** بسازی تا در پنل `/admin` کنار بخش‌های چت‌بات بنشیند. این پروژه آموزشی است و هفته‌به‌هفته به دانشجویان دوره‌ی هوش مصنوعی ارائه می‌شود؛ پس کد باید تمیز، الگومحور و قابل توضیح باشد.

## زمینه‌ی موجود (تغییرشان نده)

- جدول `leads` صف ورودی مشترک دو کانال است: فرم مشاوره‌ی سایت (`submitLead`) و تولِ `capture_lead` چت‌بات (با `conversation_id`). این جریان ورودی باید دست‌نخورده بماند.
- دسترسی دیتابیس فقط سمت سرور با service-role است (`getSupabaseAdmin`)؛ RLS روشن بدون policy. اسکیما با فایل‌های SQL دستی در پوشه‌ی `supabase/` اعمال می‌شود.
- تولید متن AI از طریق OpenRouter (`src/lib/rag/generate.ts`) با مدل قابل تنظیم از دیتابیس (`model_config`).
- پنل ادمین با کوکی HMAC تک‌رمزی (`ADMIN_PASSWORD`) محافظت می‌شود و جدول‌های خالی `admin_users` و `audit_log` از قبل برای آینده ساخته شده‌اند.
- برند: رنگ‌های pine/brass/bone/sand، فونت استعداد/وزیرمتن، کامپوننت `Logo`، اعداد فارسی با `toFa`.

## تصمیم‌های قطعی معماری

1. **جریان کلاسیک CRM**: لید ← دکمه‌ی «تبدیل» ← مخاطب (شخص) + شرکت (find-or-create از نام کسب‌وکار) + معامله‌ی اختیاری. لید تبدیل‌شده با `converted_at` و `contact_id` نشانه‌گذاری می‌شود.
2. **مراحل پایپ‌لاین config-driven** در جدول `pipeline_stages` (نه enum): جدید ← در حال بررسی ← جلسه مشاوره ← ارسال پروپوزال ← مذاکره ← بسته‌شده (موفق/ناموفق). وضعیت معامله (open/won/lost) خودکار از پرچم‌های is_won/is_lost مرحله تعیین شود.
3. **کانبان با درگ‌انددراپ بومی HTML5** — هیچ کتابخانه‌ی UI اضافه نکن؛ fallback با select برای موبایل.
4. **مبلغ معامله به تومان** (bigint) با نمایش فارسی و جداکننده‌ی هزارگان.
5. **احراز هویت چندکاربره**: فعال‌سازی `admin_users` با هش scrypt از `node:crypto` (بدون dependency جدید)، کوکی نشست امضاشده شامل نقش، ۵ نقش owner/admin/editor/operator/viewer (بیننده فقط‌خواندنی). مسیر مهاجرت نرم: تا ساخته‌شدن اولین کاربر، رمز قدیمی کار کند و فرم «ایجاد اولین مالک» فعال باشد؛ بعد از آن مسیر قدیمی بسته شود (fail-closed حتی هنگام خطای DB).
6. **audit log** برای همه‌ی عملیات مهم با ایمیل عامل — insertها حتماً await شوند (روی serverless، درخواست fire-and-forget بعد از پاسخ ممکن است گم شود).
7. نتایج AI به‌صورت **ستون روی خود موجودیت‌ها** ذخیره شود، نه جدول جدا.
8. گزارش‌ها با تجمیع در TypeScript و نمودارهای div-bar ساده — بدون کتابخانه‌ی چارت.

## ماژول‌های خواسته‌شده

### الف) هسته‌ی CRM
- **مخاطبان و شرکت‌ها** با CRUD کامل و جستجو؛ پرونده‌ی مخاطب باید **نمای ۳۶۰ درجه** باشد: تایم‌لاین یکپارچه‌ی «ثبت لید + گفتگوی کامل چت‌بات (قابل بازشدن) + فعالیت‌ها + رویدادهای تغییر مرحله‌ی معامله» مرتب از جدید به قدیم.
- **معاملات**: برد کانبان از روی `pipeline_stages`؛ جمع ارزش هر ستون؛ روزهای مانده در مرحله روی کارت؛ انتقال به مرحله‌ی باخت دلیل بگیرد؛ هر جابه‌جایی یک activity از نوع `stage_change` ثبت کند.
- **فعالیت‌ها**: تماس/جلسه/یادداشت/وظیفه با سررسید؛ وظیفه‌ی معوق قرمز هایلایت شود؛ شمارنده‌ی «وظایف معوق» به داشبورد اضافه شود. (نکته: مقدار datetime-local را سمت مرورگر به ISO تبدیل کن تا تایم‌زون سرور خرابش نکند.)
- **گزارش‌ها**: قیف تبدیل (لید ← تبدیل ← معامله ← جلسه ← برد)، ارزش پایپ‌لاین به تفکیک مرحله، منابع لید، درآمد ماهانه‌ی ۱۲ ماه با نام ماه‌های فارسی.
- **کاربران**: صفحه‌ی مدیریت کاربران با نقش، فعال/غیرفعال، ریست رمز + جدول ۱۰۰ ردیف آخر audit. (یک admin نباید بتواند owner را غیرفعال کند.)

### ب) قابلیت‌های هوش مصنوعی
1. **امتیازدهی لید** (۰–۱۰۰ + دلیل فارسی) با خروجی JSON اعتبارسنجی‌شده با zod و یک retry.
2. **خلاصه‌سازی گفتگوی چت‌بات** در پروفایل مخاطب: ۳–۵ بولت (نیازها، دغدغه‌ها، سیگنال خرید).
3. **پیشنهاد اقدام بعدی** برای معامله از روی مرحله، مدت توقف و ۵ فعالیت آخر.
4. حواست به مدل‌های reasoning باشد: سقف توکن خروجی را بالا بگیر (۲۰۰۰+) و صفحات دارای اکشن AI را `maxDuration = 60` بده.

### ج) قراردادها
- جدول `contracts` با شماره‌ی خودکار (AR-سال شمسی-شماره)، متن Markdown، مبلغ، مدت، وضعیت (draft/sent/viewed/accepted/canceled) و **share_token**.
- قالب پیش‌فرض ۱۰ ماده‌ای قرارداد مشاوره‌ی آرکان (طرفین، موضوع با متدولوژی چهار رکن، پرداخت مرحله‌ای ۴۰/۳۰/۳۰، محرمانگی، فسخ، …) که مقادیر کلاینت در آن جایگذاری شود + دکمه‌ی «بازنویسی با AI» بر اساس چالش و شناخت مشتری.
- **صفحه‌ی عمومی `/contract/[token]`** با سربرگ برند آرکان (لوگو، رنگ‌ها): کلاینت بدون لاگین ببیند، **چاپ/ذخیره PDF** کند و با واردکردن نامش **تأیید آنلاین** کند. اولین بازدید، وضعیت را به «دیده‌شده» ببرد. قرارداد تأییدشده قفل شود.
- پیام **پیگیری هوشمند**: برای قرارداد دیده‌شده‌ی تأییدنشده، AI پیام پیگیری مؤدبانه تولید کند (کپی + mailto).

### د) کمپین ایمیلی با بازبینی انسانی
- سگمنت‌های code-driven: لیدهای ازدست‌رفته، لیدهای پیگیری‌نشده (۷+ روز)، معاملات باخته، مشتریان برنده (سقف ۲۰ گیرنده‌ی دارای ایمیل).
- برای هر گیرنده AI یک ایمیل کاملاً شخصی‌سازی‌شده بنویسد (با اشاره به چالش خودش، حداکثر ۱۲۰ کلمه)؛ تولید تک‌به‌تک تا timeout نخورد؛ ادمین قبل از ارسال ویرایش/رد کند؛ ارسال با Resend (بدون کلید، تا مرحله‌ی بازبینی کار کند).

### هـ) چت با CRM
- صفحه‌ی «دستیار» + روت API استریمی: مدل با **tool-calling** (ابزارهای query_leads، query_deals، query_activities، crm_stats با zod) از Supabase پرس‌وجو کند و پاسخ تحلیلی فارسی بدهد. فقط برای ادمین واردشده. از خودش عدد نسازد.

### و) پیوند با بلاگ (فاز ۳)
- دکمه‌ی «پیشنهاد محتوا» روی لید: AI از بین پست‌های منتشرشده‌ی بلاگ مرتبط‌ترین را با چالش لید انتخاب و پیام کوتاه پیگیری بنویسد؛ اگر لید تبدیل شده، به‌عنوان یادداشت در تایم‌لاینش ثبت شود.

## الزامات عمومی

- همه‌ی سرور‌اکشن‌ها: چک نشست و نقش ← اعتبارسنجی zod ← عملیات ← audit ← revalidate؛ پیام‌های خطا فارسی و تمیز.
- SQLها idempotent (`if not exists` / `on conflict do nothing`) در فایل‌های جدا در `supabase/`.
- یک اسکریپت **داده‌ی نمایشی** (`scripts/seed-crm-demo.mjs`) با کلاینت‌های خیالی ایرانی واقع‌گرایانه بساز که همه‌ی بخش‌ها را برای دمو پر کند (شرکت‌ها، مخاطبان، معاملات در همه‌ی مراحل + برد/باخت پخش در ماه‌های گذشته، فعالیت‌های معوق/امروز/آینده، گفتگوی چت‌بات نمونه) + سوییچ `--clean`.
- ناوبری پنل را گروهی کن (داشبورد | CRM | چت‌بات | تنظیمات) بدون شکستن URLهای قبلی.
- README را با مستندات کامل CRM به‌روز کن.

## معیار پذیرش

`npm run build` سبز؛ فرم سایت ← لید ← تبدیل ← کانبان ← برد ← گزارش درآمد، سرتاسر کار کند؛ کاربر viewer هیچ تغییری نتواند بدهد؛ لینک عمومی قرارداد بدون لاگین باز شود و تأیید آنلاین وضعیت را در ادمین زنده تغییر دهد؛ بدون کلید OpenRouter/Resend همه‌جا خطا یا راهنمای فارسی تمیز نمایش داده شود، نه crash.
