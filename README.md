## چرا این معماری؟ (خلاصه برای دانشجو)

CRM بر پایه‌ی **جریان کلاسیک صنعت** (الگوی Salesforce/HubSpot) طراحی شده:

```
لید (صف ورودی)  ──تبدیل──▶  مخاطب + شرکت + معامله  ──پایپ‌لاین──▶  برد/باخت
     ▲                              │
فرم سایت / چت‌بات            فعالیت‌ها، قرارداد، کمپین، تایم‌لاین ۳۶۰°
```

- **لید** خام است و از دو کانال خودکار پر می‌شود: فرم مشاوره‌ی سایت و تولِ `capture_lead` چت‌بات.
- «تبدیل» لید، **مخاطب** (شخص)، **شرکت** (سازمان) و در صورت انتخاب یک **معامله** می‌سازد — و لید با `converted_at` نشانه‌گذاری می‌شود تا صف ورودی همیشه تمیز بماند.
- همه‌چیز حول مخاطب جمع می‌شود: **نمای ۳۶۰ درجه** یعنی تایم‌لاین یکپارچه‌ی لید + گفتگوی چت‌بات + فعالیت‌ها + تغییر مراحل معامله.

## امکانات

### 🗂 هسته‌ی CRM
| بخش | مسیر | شرح |
|---|---|---|
| لیدها | `/admin/leads` | صف ورودی، فیلتر/جستجو، تبدیل به مخاطب، خروجی CSV |
| مخاطبان | `/admin/crm/contacts` | لیست + پرونده‌ی ۳۶۰ درجه با تایم‌لاین یکپارچه |
| شرکت‌ها | `/admin/crm/companies` | ساخت خودکار از نام کسب‌وکار لید هنگام تبدیل |
| معاملات | `/admin/crm/deals` | **کانبان درگ‌انددراپ** (HTML5 بومی، بدون کتابخانه)، مراحل config-driven از جدول `pipeline_stages`، مبلغ تومانی، برد/باخت با دلیل |
| فعالیت‌ها | `/admin/crm/activities` | تماس/جلسه/یادداشت/وظیفه، سررسید، هایلایت «معوق»، کارت داشبورد |
| قراردادها | `/admin/crm/contracts` | تنظیم قرارداد برنددار از روی معامله + **لینک عمومی امن برای کلاینت** + تأیید آنلاین (امضای ساده) + چاپ/PDF |
| کمپین‌ها | `/admin/crm/campaigns` | کمپین ایمیلی سگمنت‌محور با بازبینی انسانی و ارسال Resend |
| گزارش‌ها | `/admin/crm/reports` | قیف تبدیل، ارزش پایپ‌لاین به تفکیک مرحله، منابع لید، درآمد ماهانه (تقویم فارسی) |
| کاربران | `/admin/users` | مدیریت کاربران ادمین + جدول audit log |

### ✨ هوش مصنوعی (همه با OpenRouter — مدل قابل تعویض از پنل)
1. **امتیازدهی لید** — نمره‌ی ۰ تا ۱۰۰ با توضیح فارسی؛ خروجی ساخت‌یافته با zod.
2. **خلاصه‌سازی گفتگوی چت‌بات** در پروفایل مخاطب (نیازها، دغدغه‌ها، سیگنال خرید).
3. **پیشنهاد اقدام بعدی** برای هر معامله بر اساس مرحله، مدت توقف و فعالیت‌های اخیر.
4. **پیش‌نویس قرارداد** — بازنویسی متن قرارداد متناسب با چالش و شناخت مشتری.
5. **ایمیل کمپین شخصی‌سازی‌شده** — برای هر گیرنده‌ی سگمنت، متن اختصاصی (human-in-the-loop: ادمین قبل از ارسال بازبینی می‌کند).
6. **پیام پیگیری قرارداد** — وقتی کلاینت دیده ولی تأیید نکرده.
7. **پیشنهاد محتوا** — انتخاب مرتبط‌ترین پست بلاگ (فاز ۳) برای پرورش هر لید.
8. **چت با CRM** (`/admin/crm/assistant`) — پرسش به زبان طبیعی؛ مدل با **tool-calling** (۴ ابزار پرس‌وجو) از Supabase جواب می‌گیرد. همان الگوی تولِ `capture_lead` چت‌بات در جهت عکس.

> 💡 نکته‌ی آموزشی: مدل‌های reasoning (مثل Gemini 3.5 Flash) بخشی از `maxOutputTokens` را صرف استدلال می‌کنند؛ سقف کم ⇒ خروجی JSON نیمه‌کاره. به همین دلیل سقف توابع AI بالای ۲۰۰۰ است و صفحاتی که اکشن AI دارند `maxDuration = 60` دارند. (`src/lib/crm/ai.ts`)

### 🔐 احراز هویت چندکاربره
- جدول `admin_users` با هش رمز **scrypt** از `node:crypto` (بدون کتابخانه‌ی اضافه).
- کوکی نشست: payload امضاشده با HMAC-SHA256 شامل `{uid, email, role, exp}`.
- ۵ نقش: `owner > admin > editor > operator > viewer` — بیننده در کل پنل فقط‌خواندنی است.
- **مسیر مهاجرت نرم:** تا وقتی کاربری ساخته نشده، رمز واحد قدیمی (`ADMIN_PASSWORD`) کار می‌کند و فرم «ایجاد اولین مالک» در دسترس است؛ بعد از آن فقط ایمیل+رمز (این منطق fail-closed است تا خطای DB مسیر قدیمی را دوباره باز نکند).
- **audit log نوع‌دار**: هر عملیات مهم با نام کاربر ثبت می‌شود (`src/lib/audit-actions.ts` — افزودن عملیات جدید بدون لیبل، خطای کامپایل می‌دهد).

> ⚠️ **نکته‌ی امنیتی آموزشی:** نشست‌ها stateless هستند (۳۰ روز). غیرفعال‌کردن کاربر روی کوکی‌های صادرشده‌ی قبلی اثر ندارد؛ ابطال فوری = چرخش `ADMIN_SESSION_SECRET`. پیاده‌سازی نشست DB-backed تمرین خوبی برای ادامه است.

## ساختار کد ماژول CRM

```
supabase/
  schema.sql              ← جدول leads (فاز ۱)
  chatbot-schema.sql      ← چت‌بات + جداول پایه (فاز ۲)
  crm-schema.sql          ← شرکت‌ها، مخاطبان، مراحل، معاملات، فعالیت‌ها + فعال‌سازی admin_users
  contracts-schema.sql    ← قراردادها (لینک عمومی + تأیید آنلاین)
  campaigns-schema.sql    ← کمپین‌ها و ایمیل‌های شخصی‌سازی‌شده
src/lib/
  auth.ts                 ← نشست چندکاربره + نقش‌ها (server-only)
  roles.ts                ← تعریف نقش‌ها (قابل import در کلاینت)
  passwords.ts            ← scrypt async
  audit.ts + audit-actions.ts ← ثبت رد عملیات (await شده — درس serverless!)
  crm/
    types.ts              ← تایپ‌ها و لیبل‌های فارسی
    queries.ts            ← کوئری‌های خواندنی صفحات (الگوی {data, error})
    ai.ts                 ← توابع AI + الگوی generateJson با retry
    segments.ts           ← سگمنت‌ساز کمپین‌ها (code-driven)
    reports.ts            ← تجمیع گزارش‌ها در TS
    contract-template.ts  ← قالب ۱۰ ماده‌ای قرارداد آرکان
src/app/admin/
  actions.ts              ← ورود، bootstrap، مدیریت کاربران
  crm-actions.ts          ← همه‌ی سرور‌اکشن‌های CRM (الگوی guard → zod → عملیات → audit → revalidate)
  crm/…                   ← صفحات: contacts, companies, deals, activities, contracts, campaigns, reports, assistant
src/app/contract/[token]/ ← صفحه‌ی عمومی برنددار قرارداد (بدون لاگین، توکن امن)
src/app/api/crm-chat/     ← چت با CRM (streamText + tools)
src/components/admin/crm/ ← کامپوننت‌های UI (کانبان، ۳۶۰°، ویرایشگر قرارداد، …)
scripts/seed-crm-demo.mjs ← داده‌ی نمایشی کامل برای دمو (+ --clean)
```

## راه‌اندازی

```bash
npm install
cp .env.local.example .env.local   # کلیدها را وارد کنید
npm run dev
```

### متغیرهای محیطی
```
NEXT_PUBLIC_SUPABASE_URL= / SUPABASE_SERVICE_ROLE_KEY=   # دیتابیس
ADMIN_PASSWORD=                                          # کد راه‌اندازی اولین مالک
ADMIN_SESSION_SECRET=                                    # امضای نشست (حتماً مقدار مستقل)
OPENROUTER_API_KEY=                                      # همه‌ی قابلیت‌های AI
COHERE_API_KEY=                                          # embedding چت‌بات
RESEND_API_KEY= / RESEND_FROM=                           # (اختیاری) ارسال کمپین ایمیلی
BLOG_SUPABASE_URL= / BLOG_SUPABASE_SERVICE_ROLE_KEY=     # (اختیاری) پیشنهاد محتوا از بلاگ
TELEGRAM_BOT_TOKEN= / TELEGRAM_WEBHOOK_SECRET=           # (اختیاری) کانال تلگرام
```

### دیتابیس — به این ترتیب در SQL Editor سوپابیس اجرا کنید
1. `supabase/schema.sql`
2. `supabase/chatbot-schema.sql`
3. `supabase/widget-schema.sql`
4. `supabase/crm-schema.sql`
5. `supabase/contracts-schema.sql`
6. `supabase/campaigns-schema.sql`

### داده‌ی نمایشی برای دمو
```bash
node scripts/seed-crm-demo.mjs          # ۱۰ شرکت، ۱۴ مخاطب، ۲۱ معامله، فعالیت‌ها، لیدها، گفتگوی چت‌بات
node scripts/seed-crm-demo.mjs --clean  # پاک‌سازی بعد از دمو
```

### اولین ورود
با `ADMIN_PASSWORD` وارد `/admin/login` شوید ← «ایجاد اولین حساب مالک» ← از این به بعد ورود با ایمیل+رمز و ساخت بقیه‌ی کاربران از تب «کاربران».

## سناریوی دموی کلاس (۱۰ دقیقه)

1. **لیدها** → روی یک لید تازه «✨ امتیازدهی با AI» → «تبدیل به مخاطب» (با تیک معامله + مبلغ)
2. **پرونده‌ی ۳۶۰ درجه** → «تولید خلاصه» AI روی گفتگوی چت‌بات → «+ فعالیت جدید»
3. **معاملات** → درگ کارت بین ستون‌ها → یکی را به «بسته‌شده (موفق)» ببر → «✨ اقدام بعدی؟»
4. **قراردادها** → قرارداد از روی معامله → «بازنویسی با AI» → «ارسال» → لینک را در تب ناشناس باز کن (نقش کلاینت: مشاهده، چاپ PDF، **تأیید آنلاین**) → وضعیت زنده در ادمین
5. **کمپین‌ها** → سگمنت «لیدهای پیگیری‌نشده» → «تولید همه با AI» → مقایسه‌ی متن‌های شخصی‌سازی‌شده → ویرایش یکی (human-in-the-loop)
6. **✨ دستیار** → «کدام معاملات بیش از ۱۰ روز مانده‌اند؟» → توضیح tool-calling
7. **گزارش‌ها** و **کاربران/audit log** برای جمع‌بندی
